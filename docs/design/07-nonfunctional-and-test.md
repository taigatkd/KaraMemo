# 非機能要件とテスト方針

## 1. 非機能要件

### セキュリティ

- APIキーや秘密情報をソースコードへ埋め込まない
- 不要な外部通信を追加しない
- 広告SDK以外の通信依存を持たせない
- 公式ライブラリのみを利用する

### パフォーマンス

- ローカルデータから即時に一覧表示できること
- 画面復帰時に最新状態を反映できること
- 一覧スクロールを阻害しないこと

### 保守性

- UI とデータアクセスを分離する
- バリデーションと業務ルールを ViewModel / UseCase に集約する
- テスト容易性を優先して依存を分離する

## 2. 現行 Flutter 実装との互換メモ

現行実装から確認できた互換注意点は以下。

- ピン留め状態は揮発性
- 最後に使ったアーティスト名も揮発性
- カラオケ機器設定は機器名保持のみで、詳細値の永続化は未完成
- プレイリスト削除時に曲側の `playlistId` 解除処理がない

## 3. Android 実装前の確認事項

- ピン留め状態を互換優先で揮発性にするか
- それとも `DataStore` で保持するか
- プレイリスト削除時に所属曲を自動解除するか
- 機器設定値を完全永続化するか
- 現行の検索バー終了時の挙動をそのまま再現するか

## 4. テスト方針

### 単体テスト

- モデル
- UseCase
- Repository
- ViewModel

### UI テスト

- 曲追加画面の必須入力
- 曲追加画面の重複エラー
- 曲一覧の検索
- 曲一覧の並び替え
- プレイリスト一括更新

### スモークテスト

- アプリ起動
- 下部タブ切替
- Room 初期化
- DataStore 読み書き

## 5. 重点確認項目

- 曲の重複判定
- プレイリスト名重複判定
- プレイリストまとめて追加時の差分更新
- カラオケ設定保存
- 広告のプレースホルダー切替
